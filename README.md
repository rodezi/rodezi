## How I Approach Projects

I follow a structured implementation process:

1. **Discovery** — Understand the process, users, systems, constraints, and expected outcome.
2. **Architecture** — Define the data flow, components, integrations, storage, risks, and failure handling.
3. **Implementation** — Build workflows, APIs, agents, backend logic, and integrations.
4. **Testing** — Validate success paths, failures, duplicate events, timeouts, fallbacks, and edge cases.
5. **Deployment** — Configure production environments, credentials, monitoring, and operational safeguards.
6. **Documentation and handoff** — Provide system documentation, diagrams, walkthroughs, and support procedures.
